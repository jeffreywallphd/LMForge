## Introduction
This documentation serves as a comprehensive guide to understanding, configuring, and using _LMForge_. It is organized by teams, so you can easily locate information relevant to specific areas of the project.
* **Team-Based Organization**: Each team has their own dedicated folder, allowing for easy navigation of content on a per team basis.
* **Consistency**: Documentation across the project should start to follow a standardized template to ensure uniformity and coherence. (This may vary as we establish and enact these ideas)
## How to Use This Documentation
### **1. Navigating Folders**
The documentation depository is organized into year-level directories for a multi-year approach to documentation on _LMForge_.
* `documentation/`: Core documentation applicable across all features/teams.
* `documentation/[Active Class Year]/`: Current and archived team folders.
### **2. Readme Files in Each Subfolder**
Each subfolder should contain a `README.md` file that serves as an introduction to that specific section, feature, or team. These files typically include:
* An overview of the feature or topic.
* Links to detailed documentation fiels for deeper understanding (think articles or any text that may have been instrumental in informing your team about the subject).
* A table of contents to guide you through the available documentation in that area.
Refer to these README fiels to get started quickly and navigate the content efficiently.
